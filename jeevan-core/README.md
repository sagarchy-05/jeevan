# jeevan-core

Spring Boot service — the **source of truth**. Owns users, doctors, availability and
appointments; performs booking **synchronously and transactionally**, then publishes
events to RabbitMQ and consumes the worker's results.

- **Java 21, Spring Boot 3**, Spring Web / Data JPA / Security / AMQP / Validation.
- **PostgreSQL 16** via **Flyway** migrations (`ddl-auto: validate` — schema is never
  generated by Hibernate).
- **JWT** auth (jjwt). **Swagger/OpenAPI** via springdoc.

## Run

Easiest is the root `docker compose up` (this service + its dependencies). Standalone:

```bash
# needs Postgres + RabbitMQ reachable (e.g. `docker compose up -d postgres rabbitmq`)
mvn spring-boot:run        # uses localhost defaults from application.yml
# or build the image
docker build -t jeevan-core .
```

- API base path: `/api` — Swagger UI at `/swagger-ui.html`.
- Config is env-driven (see root `.env.example`); `application.yml` ships localhost defaults.

## Key endpoints

| Method | Path | Auth | Purpose |
|--------|------|------|---------|
| POST | `/api/auth/register` | no | Register a patient |
| POST | `/api/auth/login` | no | Returns a JWT (blocked if email unverified, enabled-path) |
| GET | `/api/auth/me` | yes | Current user |
| GET | `/api/auth/verify?token=` | no | Verify email (enabled-path) |
| POST | `/api/auth/resend-verification` | no | Resend verification by `{email}` (enumeration-safe) |
| GET | `/api/doctors` | no | Paged list; `page`, `size`, `specialty`, `search` |
| GET | `/api/doctors/specialties` | no | Distinct specialties |
| GET | `/api/doctors/{id}/availability` | no | Weekly working windows |
| GET | `/api/doctors/{id}/slots?date=` | no | Dynamic 30-min slots |
| POST | `/api/appointments` | yes | Book (doctorId, startTime) — transactional |
| GET | `/api/appointments` | yes | Current patient's appointments |
| GET | `/api/appointments/{id}` | yes | Single appointment (UI polls notification status) |
| POST | `/api/appointments/{id}/cancel` | yes | Cancel own appointment |

## Layout

```
com/jeevan/core/
├── config/        SecurityConfig, RabbitMQConfig, OpenApiConfig, *Properties
├── security/      JwtService, JwtAuthFilter, AppUserDetails(Service), entry point
├── controller/    Auth, Doctor, Appointment
├── service/       Auth, Doctor, Slot, Appointment, EmailVerification
├── repository/    User, Doctor, Availability, Appointment, History, VerificationToken
├── model/         entities + enums/
├── dto/           request/ response/ (incl. ApiError envelope)
├── event/         wire payloads (booked / cancelled / notification.sent / verification)
├── messaging/     EventPublisher (after-commit), NotificationListener
└── exception/     GlobalExceptionHandler + named exceptions
```

## Notes

- **Booking concurrency** (`AppointmentService`): pessimistic doctor-row lock + the
  partial unique index; a violation → `409 SLOT_ALREADY_BOOKED`. See
  [docs/schema.md](docs/schema.md).
- **Events publish after commit** via `@TransactionalEventListener(AFTER_COMMIT)`, so a
  rolled-back booking never emits an event.
